## @DOC:MISSION-001 핵심 미션

> **"대학 데이터를 한눈에, 쉽고 빠르게"**

대학교 관리자가 Excel 데이터를 업로드하여 통계를 인터랙티브한 테이블과 차트로 시각화할 수 있는 풀스택 대시보드입니다.

### 핵심 가치 제안

#### 네 가지 핵심 가치

1. **간편함**: Excel 파일 드래그 앤 드롭 한 번으로 즉시 시각화
2. **완성도**: 플레이스홀더 없는 완전한 구현 (16 pytest 100% 통과)
3. **확장성**: Railway 클라우드 배포, PostgreSQL 기반 확장 가능 아키텍처
4. **사용자 경험**: 반응형 디자인, 로딩 상태, 명확한 에러 메시지
#### 사용자 경험 철학
- **직관성 우선**: 복잡한 매뉴얼 없이 즉시 사용 가능
- **피드백 중심**: 모든 액션에 즉각적인 시각적 피드백
- **에러 친화적**: 명확한 에러 메시지와 해결 방법 제시
- **성능 최적화**: 로딩 상태 표시, 페이지네이션으로 빠른 응답

## @SPEC:PROBLEM-001 해결할 문제

### 높은 우선순위
1. **데이터 분산 문제**: Excel 파일이 여러 곳에 분산되어 통합 관리 어려움
2. **분석 시간 과다**: 수작업 집계로 인한 시간 소요 및 오류 발생
3. **시각화 부재**: 복잡한 데이터를 직관적으로 이해하기 어려움
4. **협업 제한**: 파일 공유 방식으로 인한 버전 관리 문제

### 중간 우선순위
- 실시간 대시보드 부재로 현황 파악이 느림
- 다양한 차트 타입 필요 (막대, 선, 영역, 파이)
- 모바일 환경에서 데이터 접근 불편

### 현재 솔루션의 한계
- **Excel 기반 분석**: 수작업, 오류 가능성, 공유 어려움
- **기존 BI 도구**: 복잡한 설정, 높은 비용, 대학 특화 기능 부족

## @DOC:STRATEGY-001 차별화 요소 및 강점

### 경쟁 우위

1. **대학 특화 설계**
   - **중요한 이유**: 대학 데이터 구조에 최적화된 스키마와 카테고리
   - **구체적 사례**: 학기/학과/과목 자동 인식, 성적 분포 템플릿

2. **즉시 사용 가능**
   - **중요한 이유**: 복잡한 설정 없이 Excel 업로드 한 번으로 시작
   - **구체적 사례**: 드래그 앤 드롭 → 자동 파싱 → 즉시 시각화 (3단계)

3. **완전한 오픈소스**
   - **중요한 이유**: 커뮤니티 기여, 커스터마이징 자유
   - **구체적 사례**: MIT 라이선스, GitHub 공개, 도커 이미지 제공

4. **프로덕션 레디**
   - **중요한 이유**: 플레이스홀더 없는 완전한 구현
   - **구체적 사례**: 16개 테스트 100% 통과, Railway 배포 가능

## 💎 핵심 기능

### 1. Excel 파일 업로드 📤
- 드래그 앤 드롭 UI
- .xlsx/.xls 형식 지원
- 파일 크기 검증 (최대 10MB)
- openpyxl + pandas 자동 파싱
- 진행률 표시 및 에러 핸들링

### 2. 데이터 관리 📊
- TanStack Table v8 기반 인터랙티브 테이블
- 정렬 (오름차순/내림차순)
- 필터링 (컬럼별)
- 전체 텍스트 검색
- 페이지네이션 (10/20/50/100개)
- CRUD 연산 (생성/조회/수정/삭제)

### 3. 데이터 시각화 📈
- Recharts 기반 인터랙티브 차트
- 4가지 차트 타입: 막대/선/영역/파이
- 필드 매핑 (X축/Y축/그룹)
- 데이터 집계 (합계/평균/개수)
- 반응형 차트 디자인

### 4. 대시보드 🏠
- 통계 카드 (전체 데이터셋/레코드/크기)
- 카테고리별 분포 그리드
- 최근 업로드 목록
- 빠른 액션 버튼
